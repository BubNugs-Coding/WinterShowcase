Tidebreak

Overview

A survival/crafting game built in Godot 4.5.1 featuring a grid-based inventory system, resource gathering, item crafting, and equipment management. Survive on an island by collecting resources, crafting tools, and managing your stats.

  • Grid inventory: Tetris-style 8×6 inventory with rotatable items and container storage.
  • Crafting system: Drag-and-drop crafting with tool durability and multi-output recipes.
  • Resource gathering: Berry bushes, water sources, and driftwood piles with cooldowns.
  • Equipment: Backpacks, belts, and sashes that expand inventory and hotbar slots.
  • Survival stats: Manage Health, Hunger, Thirst, and Energy through consumables.
Game features
  • 25 unique items: Consumables, materials, tools, containers, and wearables
  • Crafting recipes: Branch + rock → stick, cloth + cloth → rope, and more
  • Tool durability: Tools take damage when used but aren't consumed
  • Storage containers: Crates (5×5 grid) and chests (10×5 grid)
  • Wearable equipment: Backpacks add 4×6 internal storage
  • Resource cooldowns: Berries (90s), water (5s), driftwood (5 uses)

Key systems

  • Crafting engine: Recipe validation, tool durability tracking, multi-output handling

  • Click: Gather resources / open containers
  • Drag: Move items / craft (drag item A onto B)
  • Right-click: Consume food/water
  • Shift+Click: Quick-transfer to/from storage

PixInsightScripts-BubNugs

GitHub repo

Overview

A PixInsight script that guides you through processing linear LRGB astrophotography data. Expects four open images named L, R, G, B.

  • Guided workflow: Step-by-step dialogs explain each processing stage.
  • Consistency: Apply the same settings (crop, calibration) across all channels.
  • Automation: Removes guesswork from the linear-to-stretched LRGB pipeline.
Workflow steps
  • Pre-flight: Image validation and reference setup
  • DynamicCrop: Crop L, then apply to RGB
  • AutoDBE: Gradient removal
  • PCC: Color calibration
  • STF → HT: Auto-stretch to histogram transform
  • LRGBCombination: Merge luminance with color
  • SCNR: Green cast removal
  • Curves/Histogram: Contrast and saturation
  • Detail enhancement: LHE, star reduction, HDRMT
  • Export: Save final image

Why it's useful

PixInsight is powerful but overwhelming. This script provides a concrete, teachable workflow that reduces the "blank page" problem when processing astrophotography data.

Setup instructions
  1. Copy LRGB_Wizard.js to your PixInsight scripts directory
  2. Refresh scripts or restart PixInsight
  3. Open your master frames named L, R, G, B
  4. Run: Script → LRGB Processing Wizard

3DWizard

GitHub repo

Overview

A client-side web toolkit for 3D printing utilities—mesh inspection, repair, conversion, and optimization. Everything runs in your browser, no uploads required.

  • Privacy-first: All processing happens locally in the browser.
  • Multiple tools: Viewer, sanity checker, repair, converter, measurement tools.
  • Modern stack: Vite + React + TypeScript + Three.js.
Available tools
  • Mesh viewer/inspector: 3D preview with stats and wireframe mode
  • Sanity checks: Detect non-manifold edges, flipped normals, open boundaries
  • Repair tools: One-click fixes for common mesh issues
  • File conversion: Convert between STL, OBJ, GLTF formats
  • Measurements: Distance, angle, and section analysis
  • Tolerance helper: Clearance recommendations for FDM/resin printing

Observatory Automation

Ongoing work connecting N.I.N.A. (Nighttime Imaging 'N' Astronomy) to the Pomfret Observatory for remote automated astrophotography.

Project timeline

  1. Initial connection attempt
    Started connecting N.I.N.A. to the observatory hardware (mount, camera, dome, focuser, filter wheel).
  2. Integration challenges
    Encountered port conflicts, driver issues, settings mismatches, and intermittent device connectivity—typical of real observatory integration work.
  3. Collaborative handoff
    Another team member took over and resolved the blocking issues.
  4. Current: Automation phase
    System is now stable and operational. Rejoining to implement automated sequencing for unattended remote operation.
Automation approach

Using N.I.N.A.'s sequencer system for reliable remote operation:

  • Basic Sequencer: Simple sequential instructions for single-target sessions
  • Advanced Sequencer: Conditional logic, triggers, loops, error recovery—essential for unattended operation
  • Safety procedures: Automated startup/shutdown, weather monitoring, altitude limits, failure recovery

What this demonstrates

  • Systems integration: Coordinating multiple hardware devices through software drivers
  • Debugging: Diagnosing failures across hardware/software boundaries
  • Collaboration: Working through handoffs and continuing complex projects
  • Automation design: Building reliable procedures for unattended operation
Technical challenges encountered
  • Port conflicts: "Could not open communications port… Error = 201"
  • Settings drift: Lat/long/elevation mismatches between systems
  • Intermittent connectivity: Devices connecting/disconnecting unpredictably
  • Multi-device coordination: Managing mount, camera, dome, focuser, filter wheel simultaneously
